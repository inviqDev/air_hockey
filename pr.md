# PR / Commit-Push Summary Notes

This file is for organizing commit-push descriptions in a short, consistent style.

Preferred format:
- Start with one short commit description line.
- Then add a numeric list of only the most important changes done since the last commit or since the start of working process.
- Each numeric point should be a one-line description.
- Do not include what is still not implemented.
- Do not include what was intentionally left unchanged.
- Do not include low-value file-by-file noise.

Use this style for future commit-push summaries in this project.
